# Private Plugin Distribution

This guide explains how the vThink Claude Code toolkit is distributed as a private plugin marketplace, and how devs install and use it.

## What is a plugin marketplace?

A plugin marketplace is a GitHub repo with a `.claude-plugin/marketplace.json` file that lists installable tools. Claude Code reads it and lets devs install skills, agents, and commands from it. The repo can be **private** — no public access needed.

The `vthink-claudecode-toolkit` repo IS the marketplace. When you install from it, you get all skills, agents, and commands in one step.

---

## Installing the toolkit

### Path A — Manual install

Use this when setting up a new machine or a new project.

Prerequisite: your GitHub access must work. Test with:
```bash
git clone git@github.com:vthinkdeveloper/vthink-claudecode-toolkit.git
```
If that works, the plugin install will too.

Then, inside any project in Claude Code:
```
/plugin marketplace add vthinkdeveloper/vthink-claudecode-toolkit
/plugin install vthink-toolkit@vthink-tools
```

All toolkit tools are now available in that project session.

### Path B — Auto-install (client projects)

When a project's `.claude/settings.json` includes the marketplace config below, Claude Code automatically prompts you to install the toolkit when you first trust the project folder. No commands needed.

```json
{
  "extraKnownMarketplaces": {
    "vthink-tools": {
      "source": {
        "source": "github",
        "repo": "vthinkdeveloper/vthink-claudecode-toolkit"
      }
    }
  },
  "enabledPlugins": {
    "vthink-toolkit@vthink-tools": true
  }
}
```

This file is generated by the `engagement-wizard` during client project setup. New devs who join the project get the tools automatically.

## Keeping tools up to date

By default, Claude Code checks for plugin updates at startup. For private repos, this requires a GitHub token:

```bash
# Add to ~/.zshrc or ~/.bashrc — one-time setup
export GITHUB_TOKEN=ghp_xxxx   # Personal access token with 'repo' scope
```

Without this, you can still update manually:
```
/plugin marketplace update vthink-tools
```

---

## Client security: locking to approved tools only

For clients who need strict control over what Claude Code tools are active, add `strictKnownMarketplaces` to the project's managed settings. This prevents devs from installing plugins from any other source.

```json
{
  "strictKnownMarketplaces": [
    {
      "source": "github",
      "repo": "vthinkdeveloper/vthink-claudecode-toolkit"
    }
  ]
}
```

Place this in the managed settings file (not `settings.json`) — individual devs cannot override it.

## vThink consultants: the engagement-kit

The `vthink-engagement-kit` is a second private marketplace containing the engagement wizard and training agent. This is for vThink consultants only — not installed into client projects.

```
/plugin marketplace add vthinkdeveloper/vthink-engagement-kit
/plugin install engagement-wizard@vthink-engagement
```

| Who | What to install |
|-----|----------------|
| Client devs | `vthink-toolkit@vthink-tools` only |
| vThink consultants | Both `vthink-toolkit@vthink-tools` + `engagement-wizard@vthink-engagement` |

## Quick reference

| I want to... | Command |
|-------------|---------|
| Add the marketplace | `/plugin marketplace add vthinkdeveloper/vthink-claudecode-toolkit` |
| Install all toolkit tools | `/plugin install vthink-toolkit@vthink-tools` |
| Update to latest version | `/plugin marketplace update vthink-tools` |
| See installed plugins | `/plugin list` |
| Validate the marketplace | `/plugin validate .` |
